## Project-Specific Rules

**Memory:**
- ! Minimize dynamic allocation everywhere
- ⊗ Use heap allocation in embedded code paths — use static pools or stack allocation only
- ~ Provide `A_NO_HEAP` compile flag to disable all heap allocation for embedded/WASM targets

**C ABI Naming Convention:**
- ! Functions and variables: `lowerCamelCase` (e.g. `aVar_setI64`, `aVar_asBool`)
- ! Types, structs, enums: `PascalCase` (e.g. `AVar`, `ABackend`, `AVarType`)
- ! Macros and constants: `UPPER_SNAKE` (e.g. `A_FLAG_OWNED`, `AVAR_MAX_BUFFER`)
- ! Namespace prefix + `_` separator for all public symbols (e.g. `aVar_`, `avar_`)
- ⊗ Use leading `_` on public ABI symbols (reserved by C standard)
- ⊗ Use double underscores `__` anywhere
- ⊗ Mix `camelCase_with_underscore` styles
- ~ Document naming convention in top-level header comment
